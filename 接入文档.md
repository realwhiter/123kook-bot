# CS2开箱模块接入文档

## 1. 模块简介

CS2开箱模块是一个独立的Python模块，用于模拟CS2游戏中的开箱功能。可以轻松集成到任何基于khl.py的KOOK Bot项目中。

## 2. 功能特性

- 📦 **收藏品管理**：支持显示、搜索、分页浏览CS2收藏品
- 🎁 **开箱模拟**：基于稀有度概率的真实开箱体验
- 🎨 **精美的卡片消息**：使用KOOK卡片消息格式，支持按钮交互
- 🔍 **搜索功能**：支持按关键词搜索收藏品
- 🎯 **按钮交互**：支持开箱、翻页、搜索、再次开箱等按钮操作
- � **状态管理**：每个用户有独立的开箱状态
- ⚡ **简单集成**：只需一个函数调用即可完成集成

## 3. 文件说明

### 3.1 核心文件

| 文件 | 说明 |
|------|------|
| `cs2_case_opener.py` | 开箱模块主文件，包含所有开箱逻辑 |
| `csqaq_collections.json` | CS2收藏品数据文件 |

### 3.2 文件依赖

- `cs2_case_opener.py` 依赖 `csqaq_collections.json` 文件
- 两个文件必须在同一目录下
- 数据文件包含CS2所有收藏品和物品的详细信息

## 5. 快速接入

### 5.1 基本接入步骤

1. 将 `cs2_case_opener.py` 和 `csqaq_collections.json` 复制到您的Bot项目目录
2. 在您的Bot主文件中导入并调用 `setup_bot_handlers` 函数
3. 启动Bot即可使用开箱功能

### 5.2 最小接入示例

```python
from khl import Bot
from cs2_case_opener import setup_bot_handlers

# 初始化Bot
bot = Bot(token='your-bot-token')

# 接入CS2开箱模块
setup_bot_handlers(bot)

# 启动Bot
bot.run()
```

### 5.3 完整接入示例

```python
import os
import asyncio
from khl import Bot
from cs2_case_opener import setup_bot_handlers

# 从环境变量获取Bot令牌
bot_token = os.environ.get('KOOK_BOT_TOKEN', 'your-bot-token')

# 初始化Bot
bot = Bot(token=bot_token)

# 接入CS2开箱模块（注册所有命令和事件处理）
setup_bot_handlers(bot)

# 启动Bot
if __name__ == '__main__':
    bot.run()
```

## 6. API说明

### 6.1 setup_bot_handlers(bot)

接入CS2开箱模块到Bot实例。

**参数**：
- `bot` (Bot): khl.py的Bot实例

**功能**：
- 注册 `/cs2` 命令
- 注册 `/open` 命令
- 注册 `/case` 命令
- 注册按钮点击事件处理
- 注册消息事件处理（搜索关键词、@机器人指令）

**示例**：
```python
from khl import Bot
from cs2_case_opener import setup_bot_handlers

bot = Bot(token='your-token')
setup_bot_handlers(bot)
bot.run()
```

### 6.2 CS2CaseOpener类

开箱模拟器的核心类。

**主要方法**：

| 方法 | 说明 | 返回值 |
|------|------|--------|
| `__init__()` | 初始化开箱模拟器 | - |
| `format_collection_list()` | 获取收藏品列表卡片 | list (卡片消息JSON) |
| `format_opening_result(name, results)` | 获取开箱结果卡片 | list (卡片消息JSON) |
| `set_search_keyword(keyword)` | 设置搜索关键词 | None |
| `next_page()` | 下一页 | bool |
| `prev_page()` | 上一页 | bool |
| `open_case(collection)` | 模拟开箱 | list (物品列表) |
| `get_collection_by_name(name)` | 根据名称获取收藏品 | dict or None |

**示例**：
```python
from cs2_case_opener import CS2CaseOpener

# 创建开箱模拟器实例
opener = CS2CaseOpener()

# 获取收藏品列表
collections = opener.format_collection_list()

# 搜索收藏品
opener.set_search_keyword("命悬一线")
results = opener.format_collection_list()

# 开箱
collection = opener.get_collection_by_name("命悬一线武器箱")
if collection:
    items = opener.open_case(collection)
    card = opener.format_opening_result(collection["name"], items)
```

### 6.3 get_user_state(user_id)

获取或创建用户状态。

**参数**：
- `user_id` (str): 用户的唯一标识符

**返回**：
- `UserState`: 用户状态对象

**示例**：
```python
from cs2_case_opener import get_user_state

# 获取用户状态
state = get_user_state("user_123")
```

### 6.4 UserState类

用户状态管理类，用于跟踪每个用户的开箱状态。

**属性**：
- `current_opener`: 当前用户的CS2CaseOpener实例
- `selected_collection`: 选中的收藏品
- `waiting_for_search`: 是否正在等待搜索输入

## 7. 命令说明

集成后，Bot将支持以下命令：

### 7.1 斜杠命令

| 命令 | 描述 |
|------|------|
| `/cs2` | 显示收藏品列表 |
| `/cs2 search <关键词>` | 搜索收藏品 |
| `/cs2 next` | 下一页 |
| `/cs2 prev` | 上一页 |
| `/cs2 open <收藏品名称>` | 开箱 |
| `/open <收藏品名称>` | 快速开箱 |
| `/case` | 显示收藏品列表 |

### 7.2 @机器人指令

| 指令 | 描述 |
|------|------|
| @Bot 开箱 | 快速打开收藏品列表 |

### 7.3 按钮交互

| 按钮 | 描述 |
|------|------|
| 开箱 | 打开指定收藏品 |
| 上一页/下一页 | 浏览收藏品列表 |
| 搜索 | 进入搜索模式 |
| 再次开箱 | 重新开箱 |

## 8. 自定义配置

### 8.1 修改稀有度概率

编辑 `cs2_case_opener.py` 中的 `RARITY_PROBABILITIES` 字典：

```python
RARITY_PROBABILITIES = {
    "普通级": 0.0,
    "军规级": 0.7992,
    "受限": 0.1598,
    "保密": 0.0320,
    "隐秘": 0.0064,
    "非凡": 0.0026
}
```

### 8.2 修改每页显示数量

在 `CS2CaseOpener.__init__` 方法中修改：

```python
class CS2CaseOpener:
    def __init__(self):
        # ...
        self.items_per_page = 5  # 修改此值
```

### 8.3 自定义稀有度颜色

修改 `_get_rarity_color` 方法：

```python
def _get_rarity_color(self, rarity: str) -> str:
    rarity_colors = {
        "普通级": "secondary",
        "军规级": "info",
        "受限": "purple",
        # 添加更多稀有度颜色映射
    }
    return rarity_colors.get(rarity, "body")
```

### 8.4 自定义稀有度表情

修改 `_get_rarity_emoji` 方法：

```python
def _get_rarity_emoji(self, rarity: str) -> str:
    rarity_emojis = {
        "普通级": "⬜",
        "军规级": "🔵",
        "受限": "🟣",
        # 添加更多稀有度表情映射
    }
    return rarity_emojis.get(rarity, "")
```

## 9. 扩展功能

### 9.1 添加新的收藏品

1. 编辑 `csqaq_collections.json` 文件
2. 按照现有格式添加新的收藏品数据
3. 重启Bot使更改生效

数据格式示例：
```json
{
    "name": "新武器箱",
    "containers": [
        {
            "short_name": "AK-47 | 血统",
            "rln": "军规级",
            "name": "AK-47 | 血统"
        }
    ]
}
```

### 9.2 自定义命令名称

在 `setup_bot_handlers` 函数中修改命令装饰器：

```python
def setup_bot_handlers(bot):
    @bot.command(name='my_custom_command')  # 修改命令名称
    async def custom_command(msg: Message, *args):
        # 您的命令处理逻辑
        pass
```

### 9.3 添加额外的命令处理

## 11. 常见问题

### Q1: 模块导入失败？

确保已安装依赖：
```bash
pip install khl.py==0.3.17
```

### Q2: 为什么有些稀有度没有颜色？

确保已为所有稀有度在 `_get_rarity_color` 方法中配置了颜色映射。

### Q3: 如何修改开箱概率？

编辑 `RARITY_PROBABILITIES` 字典，调整各稀有度的概率值。

### Q4: 如何添加新的命令？

在 `setup_bot_handlers` 函数中添加新的命令装饰器。

### Q5: 数据文件加载失败？

确保 `csqaq_collections.json` 文件与 `cs2_case_opener.py` 在同一目录下。

## 12. 注意事项

1. **文件位置**：`cs2_case_opener.py` 和 `csqaq_collections.json` 必须在同一目录
2. **依赖版本**：请使用 `khl.py==0.3.17` 以确保兼容性
3. **用户状态**：模块自动管理用户状态，无需手动处理
4. **事件处理**：模块会注册多个事件处理器，请确保在调用 `setup_bot_handlers` 前没有冲突的事件处理器
